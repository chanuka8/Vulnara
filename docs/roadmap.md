# Roadmap

This roadmap describes planned Vulnara improvements.

The project direction is safe, authorized, local-first security assessment automation.

## Completed Foundation

- Terminal CLI
- Project configuration loader
- Scope validation
- HTTP probe
- Security header scanner
- robots.txt passive reconnaissance
- sitemap.xml passive reconnaissance
- Finding rule engine
- JSON evidence storage
- HTML report generation
- Scan orchestrator
- Scan profile enforcement
- Unit tests

## Short-Term Goals

### Documentation

- Installation guide
- Usage guide
- Safety policy
- Architecture overview
- Scan profile guide
- Roadmap

### Report Improvements

- Better report styling
- Clearer finding recommendations
- Report metadata section
- Profile summary section

### CLI Improvements

- Better help messages
- Better scan summary output
- Optional verbose mode
- Cleaner error messages

## Medium-Term Goals

### Safe Web Configuration Checks

- Cookie security attribute checks
- Basic TLS configuration summary
- Redirect behavior summary
- Common security.txt discovery

### Evidence Improvements

- Evidence index file
- Scan comparison support
- Export summary as Markdown

### Test Improvements

- More integration tests
- Report template rendering tests
- Profile behavior tests

## Long-Term Goals

- Plugin-style module registration
- JSON report export
- Markdown report export
- Optional AI-assisted report explanation for local summaries
- CI workflow for automated tests

## Out of Scope

Vulnara will not include:

- Automatic exploitation
- Brute force attacks
- Password spraying
- Payload generation
- Stealth scanning
- Evasion
- Destructive testing
- Unauthorized scanning

## Project Principle

Every new feature must support authorized, non-destructive, evidence-focused security assessment.
