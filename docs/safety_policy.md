# Safety Policy

Vulnara is designed as a safe, authorized security assessment automation tool.

It is not designed for exploitation, unauthorized access, stealth activity, or destructive testing.

## Authorized Use Only

Use Vulnara only against:

- Systems you own
- Systems you administer
- Lab environments
- Training environments
- Targets where you have explicit written permission

Do not use Vulnara against third-party systems without authorization.

## What Vulnara Does

- Scope validation
- HTTP reachability checks
- Security header analysis
- robots.txt passive checks
- sitemap.xml passive checks
- Evidence storage
- Finding classification
- HTML report generation

## What Vulnara Does Not Do

- Automatic exploitation
- Brute force attacks
- Password spraying
- Credential stuffing
- Credential harvesting
- Payload generation
- Shell access
- Privilege escalation
- Persistence
- Evasion
- Stealth scanning
- Bypass techniques
- Destructive testing
- Unauthorized scanning

## Safe Defaults

- No exploit execution
- No credential attacks
- No destructive checks
- No stealth behavior
- Local evidence storage
- Explicit target authorization

## User Responsibility

The user is responsible for obtaining permission before testing, following laws and policies, keeping scan results confidential, and handling evidence responsibly.

## Project Direction

Future modules should remain non-destructive, authorized, evidence-focused, and safe by default.
